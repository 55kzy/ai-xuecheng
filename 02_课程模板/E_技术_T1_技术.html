2 批量生成：让AI自动写测试

用API批量生成测试是一个高效使用场景：

// 你的函数 function calculateDiscount(price, type) { ... } // AI生成的测试（让AI写这段） describe('calculateDiscount', () => { test('普通用户不打折', () => { expect(calculateDiscount(100, 'normal')).toBe(100) }) test('VIP用户打8折', () => { expect(calculateDiscount(100, 'vip')).toBe(80) }) test('价格为0时返回0', () => { expect(calculateDiscount(0, 'vip')).toBe(0) }) test('负数价格应该抛错', () => { expect(() => calculateDiscount(-1, 'normal')).toThrow() }) })
// 你发给AI的prompt "以下是一个函数。请帮我生成完整的单元测试，覆盖正常输入、边界条件、异常输入：\n[粘贴你的函数]"

3 辅助调试 & 文档生成

AI辅助调试

遇到bug时，把报错信息+上下文解释发给AI：

"我有以下报错： [粘贴报错信息] 相关代码： [粘贴代码片段] 我尝试过X/Y/Z但这些方法都不行。 请帮我分析可能的原因和解决方案。 注意：不要改我代码结构，只给修复建议。"

AI生成文档

"以下是以下函数的代码： [粘贴函数] 请帮我生成API文档，格式如下： - 功能描述 - 参数说明（类型、是否必须、默认值） - 返回值说明 - 使用示例 - 注意事项"

5分钟 · 建立你的AI开发工作流
明天做一次：选一个你项目中的函数，让AI帮你生成完整的测试用例和API文档。对比你手动写的和AI生成的。

进阶挑战

✅ 挑战1：建一个AI调试工作台
把你的编辑器配置成"遇到报错→自动复制报错+代码→发给AI"的快捷方式。遇到bug时一键查询。
✅ 挑战2：用AI做代码审查
把一个PR的代码变更发给AI，说"以代码审查者的角度，找出5个潜在问题"。看看AI能不能发现你漏掉的问题。
✅ 挑战3：让AI写一个CLI工具
用AI API写一个命令行小工具，比如"自动给项目生成README"或"自动格式化commit message"。跑通从AI生成到本地使用的流程。
