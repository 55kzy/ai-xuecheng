3分钟 · 先搞清楚问题在哪

你可能已经用过 AI 写东西了，但效果一般——写出来的太水、太啰嗦、用不上。

问题不在AI，在你的输入方式。

❌ 大多数人这样跟AI说
"帮我写一份周报"
✅ 换种说法效果完全不同
"我是一名运营，这周主要做了活动策划和数据分析。帮我写份周报，分本周完成、问题风险、下周计划三块。用正式的职场语气，每条控制在两行以内。"

差的 prompt → 差的输出 → 你觉得AI不靠谱。
好的prompt → 好的输出 → 你能直接拿来用，改改数字就完事。

决定输出质量的，不是哪个AI更强，而是你给了什么输入。

15分钟 · 三步让你的周报从"不能看"变"能用"
1 用角色+上下文替换"帮我写周报"

很多人上来直接"帮我写周报"，AI不知道你是谁，不知道你在什么公司，不知道你做了什么。所以它只能用最通用的 模板 给你回。

给AI说清楚三件事：

  1. 你是谁（岗位、行业）——这叫 角色设定
  2. 你做了什么（列事实、不用润色）——这叫 上下文
  3. 你要什么格式（分几块、什么语气）——这叫 结构化 指令
我是一名互联网运营，这周主要干了： 1. 周三周四跟市场部开了两次会，定了Q3活动方案 2. 整理了一版客户反馈，大概50条 3. 测试了新版登录功能，提了3个bug 4. 写了一份竞品分析，对比了A公司和B公司 5. 中间处理了一些杂事：回邮件、走审批、帮同事看问题 请按以下格式写周报： 【本周完成】 【问题与风险】（如果没有重大风险就不写） 【下周计划】 要求：语气正式但别太僵硬，每条不超过2行。
2 不满意？别重写，直接给它改的方向

很多人看到AI第一次输出不满意，就关掉重新聊。不需要。

直接在聊天里追加一句就行：

❌ 别关掉重开
"算了" → 关掉 → 重新输入 → 又生成差不多的
✅ 在同一个对话里直接说
"精简一点，保留重点就行" "写得更正式一些，给领导看的" "每条加一个数据指标"

上下文记忆 会让AI记住你的要求，顺着你的反馈调整。一次不满意、两次不满意，三次基本就是你要的样子了。

小技巧：如果你喜欢某种风格，顺便说"以后都按这个风格来"，后面它就会记住。
3 做成可复用的 prompt模板

这步很关键——做完一次后，把prompt存下来，以后每周填空就行。

存这个到备忘录：

你是一名周报助理。我告诉你我做的内容，你帮我按下面的格式整理成周报。 【本周完成】 - 每条展示关键结果和具体数字 - 每条控制在1-2行 【问题与风险】 - 优先级高的问题列前面 - 说清楚影响和需要的支持 【下周计划】 - 按优先级排列 - 标明预计完成时间 以下是我这周做的事： （每周末在这里填你的工作内容）

存到备忘录/飞书文档/微信收藏里，每周五下班前打开，填内容 → 复制粘贴 → 提交，3分钟搞定。

这节课的核心原理就三个字：给上下文

AI的局限。你给的信息越完整，它产出的东西越贴近你的需求。

差的输入好的输入
写法"帮我写周报""我是运营，这周做了X/Y/Z，按3块格式写"
结果通用 模板，用不了贴近实际，改改就能提交
耗时花5分钟改到不想改花1分钟微调数字

这个套路能迁移到哪

听懂了吗？你不是在学"用AI写周报"，你是在学一个通用的同AI协作模式：

  • 写月报/述职 — 同样结构，换更大的框架
  • 写邮件 — 说清楚收件人是谁、什么目的、什么语气
  • 写方案 — 说明背景、目标、约束条件、想要的结构
  • 分析数据 — 先给数据，再说要什么角度的分析
一个提醒：AI输出的数字和具体事件可能是编的。周报发给领导之前，把数字改对、把事实核实一遍。AI帮你的是结构和措辞，内容真实性你自己负责。

进阶挑战

✅ 挑战1：两个版本
让AI同时生成「给领导的详细版」和「抄送全组的精简版」。追加一句"再写个精简版，每条一行"就行。
✅ 挑战2：从聊天记录生成
复制本周工作群的消息发给AI："请从这些对话里提取我的工作内容，按周报格式整理" — 检验AI的提取能力。
✅ 挑战3：数据+表格
如果你的工作涉及数据，让AI把关键指标做成表格："把以上数据整理成表格，含周环比变化"

同类型迁移

三步法（角色+上下文+格式）还能用在：

  • 写会议纪要 → 给会议录音转的文字 → "请整理成会议纪要：结论、待办、责任人"
  • 写招聘JD → 给岗位信息 → "写一份JD，参考字节跳动的风格"
  • 做PPT大纲 → 给主题和页数 → "列PPT每页大纲，每页3-5个要点"

存好上面的 prompt模板，以后每次发周报流程：

1. 打开备忘录 → 粘贴本周工作内容到模板 2. 把整个prompt发给AI → 3秒出周报 3. 检查数字和事实 → 提交
