📌 AI写代码最大的坑不是"写不出来"
3分钟 · 是"写得出来但有bug你发现不了"

用AI写代码现在很普遍了。大家说"让AI写个功能，跑起来就完事"。

但问题就出在这：

  • AI写的代码能跑，但可能跑不对
  • AI写的代码功能对，但可能有严重安全隐患
  • AI写的代码表面看挺好，但可能性能差到爆炸

很多程序员把AI代码直接复制进项目，然后花一整天排查一个AI留下的隐蔽bug。还有更严重的——把含安全漏洞的代码部署到了线上。

这节课教你如何在用AI代码的同时，守住质量和安全的底线。

15分钟 · AI代码三关审查法
1 第一关：逻辑审查

AI最容易犯的逻辑错误：

常见问题AI为什么这样写怎么检查
边界条件缺失它只考虑了正常路径思考"输入为空/最大/异常时"是否处理
状态管理混乱多轮对话后它忘记之前写过的变量检查变量命名前后是否一致
使用了不存在的API它"猜"了一个函数名去文档确认API是否存在
死循环/递归无终止只写了递归逻辑没写终止条件找到所有循环和递归，检查退出条件

一个最实用的习惯：拿到AI写的代码，第一件事不是跑，而是人肉走一遍关键逻辑路径。

// AI写了一段处理用户订单的代码 // 问自己：如果用户没登录呢？如果订单为空呢？ // 如果价格为0或者负数呢？如果刚好并发请求呢？
检查技巧：把AI写的代码发回给它自己，让它"用挑剔的代码审查者角度，找出这段代码的至少3个潜在bug"。你会发现它能找出自己的问题。
2 第二关：安全审查

这是最容易被忽视但后果最严重的。

AI代码的高频安全雷区：

  1. SQL注入 — AI有时会用字符串拼接写SQL
  2. 硬编码密钥 — AI直接写死了API_KEY/AWS_SECRET
  3. 不验证用户输入 — "用户说什么我就做什么"
  4. 暴露敏感信息 — 错误信息里直接打印数据库密码
  5. 依赖过时版本 — AI引用了一个三年前不安全的库版本

审查时关注的三个关键点：

# 1. 所有用户输入来了以后有没有验证/过滤？ # 2. 有没有密钥/密码/Token硬编码在代码里？ # 3. 有没有直接操作文件/数据库/系统命令的地方？
真实案例：有人让AI写一个"用户头像上传功能"，AI生成的代码直接用了用户提供的文件名保存，并且没有类型检查——任何人都可以上传.php文件然后远程执行代码。这种漏洞放到线上就等着被黑。
3 第三关：运行验证

前两关通过后，才能让代码跑起来。

验证流程：

  1. 单元测试 — 让AI自己写测试用例，覆盖率拉到80%以上
  2. 边界测试 — 用空值、超大值、特殊字符、负数去测试
  3. 注入测试 — 输入SQL语句/脚本代码/XSS payload看是否被过滤
  4. 性能测试 — 问AI"这个方案在大数据量下性能如何？瓶颈在哪？"
我只需要用来做测试，需要简洁的测试框架样例 // 让AI帮你写测试： // "这是我上一个回复里写的函数，帮我对它写单元测试， // 覆盖正常输入、边界输入、异常输入三个场景"
好习惯：每次用AI写的代码前，花5分钟跑一遍这三关。前三次你可能不习惯，十次之后就是肌肉记忆了。
💡 理解为什么
5分钟 · 为什么AI写的代码容易有这些问题

AI写代码的本质：模式匹配，不是理解

AI不是真的"理解"你的需求然后去实现。它是在海量代码里匹配最相似的模式然后拼接出来。

  • 它见过一万段安全的登录代码，也见过一千段有SQL注入的登录代码
  • 它的训练数据里既有最佳实践，也有反模式
  • 它不知道哪段代码会被部署到线上，哪段会被攻击
  • 它不会主动说"这个实现有安全风险"

跟已有课程的区别

  • 已有E·P prompt实战课 — 教你怎么给AI写好的prompt来生成代码（侧重输入）
  • 已有E·K 工具选型课 — 教你选什么AI工具/平台来辅助开发（侧重工具）
  • 这节E·L 代码核实课 — 教你验证AI生成的代码质量（侧重输出）

三门课构成技术人的AI工作流闭环：**输入 → 工具 → 输出验证**。缺了这一环，前面两环做得再好也可能翻车。

核心原则：把AI当成"写初稿的实习生"，不是"提交代码的正式员工"。你永远要对AI的代码负最终责任。

5分钟 · 建立你的AI代码审查习惯
明天做一次：下次让AI生成一段代码后，故意不做审查直接跑。看看跑的过程中有没有出现边界崩溃。然后对照三关审查法检查，找到AI写的潜在问题。

进阶挑战

✅ 挑战1：让AI攻击自己的代码
把AI写的一段代码发回给它，说"请以安全审计员的身份，找出这段代码里的所有安全漏洞"。看它能找到几个。
✅ 挑战2：让AI补全缺少的验证
把AI生成的有安全问题的代码发回给它，说"给这段代码加上所有必要的输入验证和安全检查"。看第二版是否还漏了什么。
✅ 挑战3：对比两个AI的输出质量
同一个需求让两个不同AI生成实现，从正确性、安全性、可读性三个维度对比。你会发现质的差距。

⚡ 记住一句话

**AI代码 = 实习生写的代码，不审查别merge**

三关审查法（5分钟）： 1. 逻辑审查 → 边界/状态/假API/死循环 2. 安全审查 → SQL注入/硬编码/不验证输入/暴露信息 3. 运行验证 → 单元测试/边界测试/安全测试/性能测试
