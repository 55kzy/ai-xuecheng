📌 你让AI写代码，AI是不是总给你"能用但不好用"的代码？
3分钟 · 为什么你以为写得很清楚的prompt，AI就是听不懂

你让AI"帮我写个查询函数"，它给你一个SELECT * FROM users WHERE id = 1。

你让AI"优化一下这段代码"，它给你把两个for循环改成了列表推导式，性能提升1%，但可读性下降了50%。

不是你表达能力差，是你还在用跟人沟通的方式跟AI说话。

写代码的prompt跟日常对话的prompt完全是两码事。代码prompt需要包含：

  • 具体的技术约束（框架、语言版本、性能要求）
  • 明确的输入输出
  • 边界情况和错误处理
先看一个对比：

❌ 差输入（就像你在Slack上问同事）：
帮我写个函数，处理用户上传的CSV文件。
AI反馈：写了一个简单的read_csv，不处理编码、不验格式、没有错误处理

✅ 好输入（像在写技术需求文档）：
写一个Python函数 process_csv(file_path: str) -> pd.DataFrame 要求： 1. 自动检测编码（UTF-8/GBK） 2. 如果文件不是CSV格式，抛出自定义异常 3. 处理空行和列数不匹配的行 4. 返回DataFrame并打印 rows_processed 和 rows_skipped 5. 加类型标注和docstring 只用标准库 + pandas
AI反馈：生成了一段几乎可以直接上生产的代码，包含类型标注、异常处理、日志输出

核心差异：差输入在"表达需求"，好输入在"写技术规约"。你给AI的信息量决定了它给你的代码质量——给多给少，回报完全不同。

✓ 写出让AI一次生成可用代码的prompt，减少返工
✓ 能精准描述技术约束（框架、版本、性能指标）

🎯 动手实操
15分钟 · 三步写出AI一次搞懂的代码prompt
1 套用"技术模板"——别再只说需求了

写代码prompt套这个模板，别偷懒：

语言和框架：Python 3.11 + FastAPI 任务描述：写一个POST接口，接收用户上传的JSON 输入：{user_id: string, items: array} 输出：{status: string, created_at: string} 技术约束：用Pydantic做类型校验，错误信息用中文 边界情况：user_id为空时返回400， items为空数组时返回200但status为"empty" 不要用：同步IO以外的第三方库

模板结构：环境 + 任务 + 输入/出 + 约束 + 边界 + 禁忌

你不是在跟AI"聊天"，你是在给它写一个简短的PRD。越接近代码规范描述，AI给你的越接近可用代码。

2 三个场景的prompt备忘录

不同场景要换不同的prompt策略，分享三个你马上就能用的：

🔧 场景一：写新函数
用第1步的"技术模板" + 加一句"写完顺便写3个单元测试"。这样才能确保AI考虑了边界情况和返回值。
🔄 场景二：重构代码
先贴代码，再说：
"这段代码的目标是{xxx}。请在不改变核心逻辑的前提下，提升可读性和性能。每处改动说明理由。"
注意：一定要告诉AI"不能改目标"——不然它会把你的业务逻辑也改掉。
🐛 场景三：调试bug
先贴报错信息，再说：
"这个报错可能的原因有哪些？按可能性从高到低列出。把你最怀疑的那个原因对应的解决方案写出来，其他原因只写排查方法。"
这样AI不会只给一种可能性，你能系统地排查。
3 验证——别直接粘代码就跑

AI给的代码，就算看起来再完整，也按这个流程验证：

1. 只读不改：先通读一遍，看逻辑对不对 2. 检查API：如果调了第三方库，确认API签名没错 3. 逐步执行：如果是核心逻辑，在REPL里一行一行跑 4. 看异常路径：传个空值/边界值进去 5. 最多改3行：如果超过3行需要改，重建prompt重来吧

为什么最多只改3行？因为超过3行说明AI没有理解你的需求，硬改出来的代码后面会有更多bug。重建prompt加上上一步的教训，重来一次通常比硬改快。

💡 理解为什么
5分钟 · 你给AI的信息量 = AI给你的代码质量

核心原理：代码prompt本质上是一份"微型技术文档"

非技术人写prompt像是在指方向（"那里有块肉"），技术人写代码prompt应该像在画图纸（"长20cm宽15cm厚3cm，铁质"）。

为什么你的prompt要包含环境、约束、边界？因为AI的工作方式是：它见过的代码越多，给出来的就越符合那个模式。你给的约束越精确，它就越能匹配到"这个需求+这个约束"对应的最佳代码模式。

一个数据点：把同一个需求写成"帮我写个函数"（6个字）和写成"技术模板"（约100字），AI生成的代码通过率从约30%提升到约80%。不是AI变强了，是你给它提供了足够的信息来理解你的需求。

这个能力能迁移到哪

  • 写技术方案 — 给AI一个"环境+目标+约束"的模板，它能生成80%初稿
  • 写技术文档 — 让AI根据你的代码和注释生成README和API文档
  • 做技术选型 — 用"技术模板"的方式让AI比较不同方案，比搜博客高效

进阶挑战

✅ 挑战1：存一份个人代码prompt模板库
创建一个"代码prompt模板.md"文件，把你常用的几个场景（写API、写工具函数、写测试、写SQL）的prompt模板存进去。每次写代码前对着模板填参数。
✅ 挑战2：让AI写一个完整的单元测试
找一个你最近写的函数（50行以上），用"技术模板"写好prompt，让AI生成完整的单元测试。要求覆盖：正常路径、边界值、异常输入、性能指标各一条。
✅ 挑战3：让AI重构一段代码并解释理由
找一段你已经写好的代码（100-200行），用重构prompt让AI优化。然后逐条看它的理由，判断每条是否有道理。这是最好的学习"怎么写出更好代码"的方式。

这节课只是个开始

掌握了基础的代码prompt模板，你还有很长的路可以走：

  • 大型重构 — 不只是单个函数，而是跨文件的模块级重构
  • 架构级prompt — 让AI理解你的项目架构，在正确的位置做改动
  • 代码审查自动化 — 让AI做第一轮CR，你只看AI标记的问题

这些在T2进阶课里会继续深入——不只是"写一个函数"的教学，而是让你在真实项目中高效使用AI开发。你现在学会的模板，就是未来自动化工作流的起点。
