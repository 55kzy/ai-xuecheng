# 自适应出题引擎 — 设计草案

## 输入
- 客户方向选择 (1-2个, 来自7大方向)

## 状态维护 (客户画像)
```json
{
  "directions": ["C"],
  "scores": { "K": null, "P": null, "L": null, "T": null, "C": null },
  "confidence": 0,
  "askedQuestions": [],
  "usedTypes": [],
  "currentLevel": 1,
  "questionCount": 0
}
```

## 出题策略 (伪代码)

```
def selectNext(customerProfile):
    # 1. 检查终止条件
    if questionCount >= 6: return STOP
    if confidence >= 0.8 and scoredDims >= 3: return STOP
    if questionCount >= 3 and any keyDim is null: forceAskThatDim
    
    # 2. 选题优先级
    if confidence < 0.3:  # 初期先定位
        pick: type=H or P (习惯/偏好, 低压暖场)
        dim = K (工具认知, 最基础)
    elif scoredDims < 2:  # 缺维度
        pick: type not in usedTypes, dim = null dim
    elif direction needs refinement:
        pick: type=G (梯度, 精确定位水平)
    else:  # 补全
        pick: type not in usedTypes, dim = lowest confidence dim
    
    # 3. 从题库筛选符合条件的题
    candidates = filter(题库, 
        direction overlaps customer.directions,
        type == pick.type,
        dimension == pick.dim,
        id not in customer.askedQuestions
    )
    return random(candidates)
```

## 题型轮换规则
- 同客户不重复题型
- 6种题型 G/S/P/T/H/C 用完为止
- 最多6题, 最少3题

## 方向选择策略 (选2个方向时)
- 包含G(全面了解) → 以另一个方向为主
- 两个具体方向 → 走交叉题库
- 无交叉题库 → 各出1题后, 引擎决定主攻方向
