# AI学程 出题规则 (v1.0)

## 核心原则（2026-05-09 确认）

### 1. 真实反映
每道题的每个选项都要能反映真实情况——不是编造场景，
而是现实中不同用户真的会那样做。

### 2. 避免明显优劣
选项不能有明显的好/坏之分，不能有"傻子→还行→不错→高手"的阶梯感。
否则客户会下意识选"好"选项，测不出真实水平。

### 3. 像"不同人的不同做法"
每个选项看起来都像正常人会做的选择，
但选项本身能透露出用户的信息素养/使用习惯/认知深度。

### 4. 文字简洁
题干能一句话说清楚就不要两句。选项用口语，不用书面语。

### 5. 动态出题
一问一答，根据当前回答决定下一题出什么。
每道题独立，不影响其他题的出题逻辑。

## 好题 vs 差题对比

### ❌ 差（太明显，客户会选最高分项）
```
题干：AI说某只股票会涨，你会怎么做？
→ 直接买入           (0分) ← 傻子
→ 先查查AI为什么      (1分) ← 还行
→ 只当参考           (2分) ← 不错
→ 完全不用AI做投资   (3分) ← 高手
```
问题：每个选项都能看出分数高低，客户会选最后一个。

### ✅ 好（每个选项都是真实反应）
```
题干：AI说某只股票会涨，你会怎么做？
→ 先看看它说的靠不靠谱再说   (0分)
→ 用它说的信息自己再去查查   (1分)
→ 参考一下，但不会完全听它的  (2分)
→ 让它整理信息我自己做判断   (3分)
```
优点：每个选项都是不同人的真实做法，没有明显对错。

## 题型的处理建议

| 题型 | 说明 | 注意事项 |
|------|------|---------|
| G 梯度 | 测量使用深度/熟练度 | 选项要像"自然的使用方式"，不要像"水平测试" |
| S 情景 | 给定场景问怎么做 | 每个选项要是合理但不同的应对方式 |
| P 偏好 | 问倾向性 | 天然好选择，选项就是不同偏好 |
| T 两难 | 进退两难的取舍 | 选项之间真有取舍，没有完美选项 |
| H 习惯 | 问实际做了什么 | 最好，客户说的就是真实情况 |
| C 概念 | 问理解程度 | 用大白话描述，不用教科书语言 |

## 常用避坑技巧

1. **最后一个选项不用写最长最详细**
   → 改前："完整的prompt应该包含system+few-shot+格式+边界...完整操作"
   → 改后："看情况，复杂的事就多写点"

2. **选项分数不要跟文字长度挂钩**
   → 0分词也可以是有道理的选项

3. **倾向题和习惯题 > 概念题**
   → 问"你实际怎么做的"比"你认为什么是对的"更能反映真实水平

4. **如果客户能猜到这是"测水平"，那设计就有问题**
